    1. Missing External JS Files (P0 - CRITICAL)¶

    Статус: ✅ ИСПРАВЛЕНО

    Проблема: 

    file:///apiClient.js - net::ERR_FILE_NOT_FOUND
file:///SyncManager.js - net::ERR_FILE_NOT_FOUND
file:///errorBoundary.js - net::ERR_FILE_NOT_FOUND

    Причина: Абсолютные пути /apiClient.js не работают при загрузке через file:// протокол.

    Решение: 

    <!-- Было -->
<script src="/apiClient.js"></script>

<!-- Стало -->
<script src="./apiClient.js"></script>

    Файлы: - index.html:11585-11591

    Результат: ✅ Все внешние JS файлы загружаются корректно через HTTP ✅ APIClient, SyncManager, ErrorBoundary инициализируются успешно

    2. QuoteCalc is not defined (P0 - CRITICAL)¶

    Статус: ✅ ИСПРАВЛЕНО

    Проблема: 

    ReferenceError: QuoteCalc is not defined
at index.html:11534 (beforeunload handler)

    Причина: Использование глобальной переменной QuoteCalc вместо window.quoteCalc.

    Решение: 

    // Было
window.addEventListener('beforeunload', function() {
    if (QuoteCalc.currentRegion) {
        QuoteCalc.saveCatalogToRegion(QuoteCalc.currentRegion);
    }
});

// Стало
window.addEventListener('beforeunload', function() {
    if (window.quoteCalc && window.quoteCalc.currentRegion) {
        window.quoteCalc.saveCatalogToRegion(window.quoteCalc.currentRegion);
    }
});

    Файлы: - index.html:11532-11542

    Результат: ✅ Нет ошибок при закрытии страницы ✅ Автосохранение работает корректно

    ⚠️ ОБНАРУЖЕННЫЕ НОВЫЕ ПРОБЛЕМЫ¶

    1. JSON Double Serialization (P2 - MEDIUM)¶

    Статус: ⚠️ НЕКРИТИЧНО (не блокирует работу)

    Проблема: 

    SyntaxError: Unexpected token '"', ""a90319fc00f7"" is not valid JSON

    Причина: Клиент отправляет уже сериализованную строку, а body-parser пытается её парсить снова: - Клиент отправляет: JSON.stringify("a90319fc00f7") → ""a90319fc00f7"" - Сервер ожидает: "a90319fc00f7"

    Влияние: - ❌ Запросы к SQLite БД падают с 400 Bad Request - ✅ localStorage продолжает работать нормально - ✅ Пользователь не теряет данные

    Рекомендация: Исправить в будущей версии. Найти места где используется двойная сериализация: 

    // Неправильно
JSON.stringify(JSON.stringify(id))

// Правильно
JSON.stringify(id)

    Приоритет: P2 (Средний) - не блокирует основной функционал
