    Работа со сметами¶

    Версия: 2.3.0 Дата обновления: 6 ноября 2025

    Введение¶

    Этот раздел описывает все аспекты работы со сметами в Quote Calculator: от создания до печати и экспорта. Вы узнаете все возможности системы для эффективной работы с коммерческими предложениями.

    Содержание раздела¶

    1. Создание смет¶

    Узнайте, как создавать сметы различными способами: - С нуля (пустая смета) - Из шаблона - Дублированием существующей - Импортом из файла

    Время чтения: 10 минут

    2. Редактирование смет¶

    Полное руководство по редактированию: - Изменение данных клиента и группы - Добавление и удаление услуг - Массовое редактирование - Работа с категориями

    Время чтения: 15 минут

    3. Расчёты и наценки¶

    Подробно о механике расчётов: - Индивидуальные наценки - Скрытая наценка - НДС - Расчёт прибыли и маржи - Формулы

    Время чтения: 12 минут

    4. Печать и экспорт¶

    Все способы вывода смет: - Печать на принтер - Сохранение в PDF - Экспорт в JSON - Форматирование для клиента

    Время чтения: 8 минут

    Быстрые ссылки¶

    Частые задачи¶

    Задача Инструкция
    Создать новую смету Ctrl+N → Заполнить данные → Сохранить
    Загрузить смету Ctrl+O → Выбрать из списка
    Добавить услугу Ctrl+K → Выбрать из каталога
    Изменить наценку Клик на поле → Ввести % → Enter
    Распечатать Ctrl+P → Настроить → Печать
    Сохранить Ctrl+S (автосохранение каждые 30 сек)

    Горячие клавиши¶

    Действие Клавиша
    Новая смета Ctrl+N
    Открыть Ctrl+O
    Сохранить Ctrl+S
    Печать Ctrl+P
    Каталог Ctrl+K
    Удалить услугу Delete

    Жизненный цикл сметы¶

    1. Создание
   ↓
2. Заполнение данных клиента
   ↓
3. Добавление услуг
   ↓
4. Настройка наценок
   ↓
5. Проверка расчётов
   ↓
6. Сохранение (Ctrl+S)
   ↓
7. Отправка клиенту (PDF)
   ↓
8. [Опционально] Редактирование
   ↓
9. [Опционально] Архивирование

    Типы смет¶

    Черновик (Draft)¶

    Статус: В работе
Назначение: Предварительный расчёт
Действия: Можно менять всё

    Отправлено клиенту (Sent)¶

    Статус: Ожидание ответа
Назначение: КП отправлено
Действия: Можно редактировать с версионированием

    Утверждено (Approved)¶

    Статус: Принято клиентом
Назначение: Финальная версия
Действия: Только просмотр (лучше не менять)

    Архив (Archived)¶

    Статус: Завершено/отменено
Назначение: Хранение истории
Действия: Только просмотр

    Лучшие практики¶

    Именование смет¶

    Рекомендуемый формат: 

    {Клиент}_{Дата}_{PAX}pax_{ID}

Примеры:
ivanov_20251215_10pax_est_001
petrov-company_20251220_25pax_est_002

    Система именует автоматически: 

    filename = `${transliterate(clientName)}_${date}_${pax}pax_${id}.json`

    Организация¶

    Папки/теги (в будущих версиях): 

    /estimates
  /active          # Активные сметы
  /approved        # Утверждённые
  /archive         # Архив

    Сейчас: Используйте префиксы в именах 

    ACTIVE_ivanov_20251215...
APPROVED_petrov_20251220...
ARCHIVED_sidorov_20251201...

    Версионирование¶

    При изменениях: 

    1. Сохранить текущую версию
2. Дублировать смету
3. Изменить копию
4. Добавить суффикс: _v2, _v3

    Пример: 

    ivanov_20251215_10pax_est_001.json     # v1
ivanov_20251215_10pax_est_001_v2.json  # v2 (после правок)

    Валидация данных¶

    Обязательные поля¶

    • ✅ Имя клиента
    • ✅ Телефон
    • ✅ Email
    • ✅ PAX (≥ 1)
    • ✅ Хотя бы одна услуга

    Предупреждения¶

    ⚠️ Система предупредит: - Отсутствуют обязательные поля - PAX = 0 - Нет услуг в смете - Цена услуги = 0 - Отрицательные значения

    Ошибки¶

    ❌ Система не даст сохранить: - Пустое имя клиента - Некорректный email - PAX < 1

    Автосохранение¶

    Как работает¶

    1. Каждые 30 секунд → Сохранение в localStorage
2. При изменениях → Trigger автосохранения
3. При закрытии → Финальное сохранение

    Индикаторы¶

    💾 Сохранено           # Последнее изменение сохранено
⏳ Сохранение...       # Идёт процесс сохранения
⚠️ Не сохранено       # Есть несохранённые изменения

    Ручное сохранение¶

    Нажмите Ctrl+S для: - Сохранения на сервер (не только localStorage) - Создания backup - Гарантированного сохранения

    Резервные копии¶

    Автоматические backups¶

    Когда создаются: 

    1. При каждом сохранении на сервер (Ctrl+S)
2. Перед критическими изменениями
3. Перед удалением

    Хранение: 

    В таблице backups с timestamp
Доступ: Меню → Backups → История

    Восстановление¶

    1. Меню → Backups
2. Выбрать версию (по дате/времени)
3. Просмотреть изменения
4. Нажать "Восстановить"
5. Подтвердить

    Поиск и фильтрация¶

    Глобальный поиск (Ctrl+F)¶

    Поиск по: - Имени клиента - Email - Телефону - ID сметы - Содержимому (услуги)

    Фильтры¶

    Статус: [Все / Черновик / Отправлено / Утверждено]
Дата: [Последние 7 дней / 30 дней / Все время]
PAX: [1-5 / 6-10 / 11-20 / 21+]
Стоимость: [До $1000 / $1000-$5000 / $5000+]

    Сортировка¶

    По дате создания ↓ (по умолчанию)
По дате изменения
По имени клиента
По стоимости

    Экспорт и импорт¶

    Экспорт¶

    Форматы: 

    // JSON (полный экспорт)
{
  "version": "2.3.0",
  "client": {...},
  "services": [...],
  "calculations": {...}
}


    // PDF (для клиента)
Только видимая информация
Форматирование для печати

    Импорт¶

    Поддерживаемые форматы: - JSON v1.0.0 (legacy) - JSON v1.1.0 (current) - CSV (только каталоги)

    Валидация при импорте: 

    ✅ Проверка версии
✅ Проверка структуры
✅ Валидация данных
⚠️ Предупреждение при несовместимости

    Совместная работа¶

    Текущая версия (v2.3.0)¶

    ❌ Не поддерживается: - Одновременное редактирование - Real-time синхронизация - Комментарии - История изменений (кроме backups)

    ✅ Работает: - Последовательное редактирование - Экспорт/импорт для обмена - Backups для версионирования

    Workaround для команды¶

    1. Один человек → Владелец сметы
2. Экспорт → JSON файл
3. Отправить коллеге
4. Коллега импортирует
5. Вносит изменения
6. Экспортирует обратно
7. Владелец импортирует финальную версию

    Производительность¶

    Рекомендации¶

    Оптимально: - До 100 услуг в смете - До 1000 смет в базе - Файлы до 5MB

    Если тормозит: 

    1. Удалить неиспользуемые услуги
2. Архивировать старые сметы
3. Очистить localStorage
4. Обновить браузер

    Ограничения¶

    // Текущие лимиты
maxServicesPerEstimate: 1000
maxEstimates: 10000
maxFileSize: 5MB
maxNameLength: 100

    Безопасность¶

    Права доступа¶

    Текущая версия: 

    Нет встроенной аутентификации
Все пользователи имеют полный доступ

    Рекомендации: 

    1. Не открывать порт в интернет
2. Использовать Nginx с basic auth
3. VPN для удалённого доступа
4. Регулярные backups

    Конфиденциальность¶

    Данные клиентов: - Хранятся локально в базе данных - Не передаются третьим лицам - Шифрование в покое не реализовано

    Рекомендации: 

    1. Шифрование диска (FileVault, BitLocker)
2. Резервные копии в зашифрованном виде
3. Безопасное удаление старых данных
