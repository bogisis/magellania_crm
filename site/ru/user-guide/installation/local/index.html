    Локальная установка¶

    Версия: 2.3.0 Дата обновления: 6 ноября 2025

    Введение¶

    Локальная установка - это самый простой способ запустить Quote Calculator на вашем компьютере для личного использования или разработки. Вся установка занимает 5-10 минут.

    Системные требования¶

    Перед началом убедитесь, что у вас установлено:

    • ✅ Node.js версии 16.x или выше
    • ✅ npm версии 8.x или выше
    • ✅ Git (для клонирования репозитория)

    Проверка установленных версий¶

    # Проверить Node.js
node --version
# Ожидается: v16.x.x или выше

# Проверить npm
npm --version
# Ожидается: 8.x.x или выше

# Проверить Git
git --version
# Ожидается: git version 2.x.x или выше

    Если что-то не установлено, см. Системные требования.

    Пошаговая установка¶

    Шаг 1: Скачать проект¶

    # Клонировать репозиторий
git clone https://github.com/your-org/quote-calculator.git
cd quote-calculator

    Или скачайте архив вручную: 1. Перейдите на https://github.com/your-org/quote-calculator 2. Нажмите "Code" → "Download ZIP" 3. Распакуйте архив 4. Откройте терминал в папке проекта

    Шаг 2: Установить зависимости¶

    npm install

    Что происходит: - Устанавливаются все необходимые Node.js пакеты - Создается папка node_modules/ - Процесс занимает 1-2 минуты

    Ожидаемый вывод: 

    added 245 packages, and audited 246 packages in 45s

35 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

    Шаг 3: Запустить сервер¶

    npm start

    Что происходит: - Запускается Express.js сервер на порту 4000 - Инициализируется SQLite база данных - Создаются необходимые папки (estimate/, catalog/, backup/, logs/)

    Ожидаемый вывод: 

    [2025-11-06 07:00:00] INFO: Server starting...
[2025-11-06 07:00:00] INFO: Storage type: sqlite
[2025-11-06 07:00:00] INFO: Database initialized
[2025-11-06 07:00:00] INFO: Server running on http://localhost:4000

    Шаг 4: Открыть в браузере¶

    Откройте браузер и перейдите по адресу:

    http://localhost:4000

    Вы должны увидеть главную страницу Quote Calculator!

    Структура проекта¶

    После установки у вас будет следующая структура:

    quote-calculator/
├── server-with-db.js        # Основной сервер
├── package.json             # Зависимости и скрипты
├── db/
│   ├── quotes.db           # SQLite база данных
│   └── migrations/         # Миграции базы данных
├── estimate/               # JSON файлы смет (legacy)
├── catalog/                # JSON файлы каталогов (legacy)
├── backup/                 # Автоматические backups
├── logs/                   # Логи сервера
│   ├── combined.log
│   └── error.log
├── index.html             # Frontend приложение
├── styles.css             # Стили
└── docs/                  # Документация

    Конфигурация¶

    Переменные окружения¶

    Создайте файл .env в корне проекта для настройки:

    # .env
PORT=4000
STORAGE_TYPE=sqlite
APP_ENV=development
JSON_LIMIT=50mb
DB_PATH=./db/quotes.db
LOG_LEVEL=info

    Параметры:

    Параметр Значение по умолчанию Описание
    PORT 4000 Порт сервера
    STORAGE_TYPE sqlite Тип хранилища (sqlite/file)
    APP_ENV development Окружение (development/production)
    JSON_LIMIT 50mb Лимит размера JSON запросов
    DB_PATH ./db/quotes.db Путь к базе данных
    LOG_LEVEL info Уровень логирования (error/warn/info/debug)

    Изменение порта¶

    Если порт 4000 занят, измените его:

    # Вариант 1: через .env файл
echo "PORT=3000" > .env
npm start

# Вариант 2: через переменную окружения
PORT=3000 npm start

    Использование FileStorage (legacy)¶

    Если нужно использовать файловое хранилище вместо SQLite:

    STORAGE_TYPE=file npm start

    Внимание: FileStorage устарел и будет удален в будущих версиях. Рекомендуется использовать SQLite.

    Тестирование API¶

    # Получить список смет
curl http://localhost:4000/api/estimates

# Получить список каталогов
curl http://localhost:4000/api/catalog/list

# Экспортировать все данные
curl http://localhost:4000/api/export/all > backup.json

    Управление сервером¶

    Запуск сервера¶

    # Development mode с логами
npm start

# Production mode
NODE_ENV=production npm start

# В фоновом режиме (macOS/Linux)
nohup npm start > server.log 2>&1 &

# В фоновом режиме (Windows)
start /B npm start

    Остановка сервера¶

    # Ctrl+C в терминале, где запущен сервер

# Или найти процесс и убить его (macOS/Linux)
lsof -i :4000 | grep LISTEN
kill -9 <PID>

# Или найти процесс и убить его (Windows)
netstat -ano | findstr :4000
taskkill /PID <PID> /F

    Перезапуск сервера¶

    # Остановить (Ctrl+C)
# Затем запустить снова
npm start

    Обновление¶

    Способ 1: Git pull¶

    # Сохранить данные
mkdir -p backup/$(date +%Y%m%d)
cp db/quotes.db backup/$(date +%Y%m%d)/
cp -r estimate/ backup/$(date +%Y%m%d)/
cp -r catalog/ backup/$(date +%Y%m%d)/

# Остановить сервер (Ctrl+C)

# Обновить код
git pull origin main

# Установить новые зависимости
npm install

# Запустить сервер
npm start

    Способ 2: Скачать новую версию¶

    # Сохранить данные
cp db/quotes.db ~/backup/
cp -r estimate/ ~/backup/
cp -r catalog/ ~/backup/

# Скачать новую версию с GitHub
# Заменить файлы проекта

# Восстановить данные
cp ~/backup/quotes.db db/
cp -r ~/backup/estimate/ ./
cp -r ~/backup/catalog/ ./

# Установить зависимости
npm install

# Запустить сервер
npm start

    Миграция данных¶

    Из FileStorage в SQLite¶

    Если вы использовали старую версию с FileStorage:

    # Запустить миграцию
node db/migrations/migrate-to-db.js

# Проверить результат
sqlite3 db/quotes.db "SELECT COUNT(*) FROM estimates"

    Что мигрирует: - Все сметы из estimate/*.json - Все каталоги из catalog/*.json - Все backups из backup/*.json

    Из SQLite в другую установку¶

    # Экспортировать все данные
curl http://localhost:4000/api/export/all > full-export.json

# На новой установке импортировать
curl -X POST http://localhost:4000/api/import/all \
  -H "Content-Type: application/json" \
  -d @full-export.json

    Резервное копирование¶

    Автоматические backups¶

    Система автоматически создает backup при каждом сохранении сметы в таблице backups.

    Ручное резервное копирование¶

    # Способ 1: Экспорт всех данных в JSON
curl http://localhost:4000/api/export/all > backup-$(date +%Y%m%d).json

# Способ 2: Копия базы данных SQLite
cp db/quotes.db backup/quotes-$(date +%Y%m%d).db

# Способ 3: Экспорт бинарной копии БД через API
curl http://localhost:4000/api/export/database > backup-$(date +%Y%m%d).db

    Восстановление из backup¶

    # Из JSON
curl -X POST http://localhost:4000/api/import/all \
  -H "Content-Type: application/json" \
  -d @backup-20251106.json

# Из копии БД
cp backup/quotes-20251106.db db/quotes.db
npm start

    Удаление¶

    Полное удаление¶

    # Остановить сервер (Ctrl+C)

# Сохранить данные (если нужно)
mkdir ~/quote-calculator-backup
cp db/quotes.db ~/quote-calculator-backup/
cp -r estimate/ ~/quote-calculator-backup/
cp -r catalog/ ~/quote-calculator-backup/

# Удалить проект
cd ..
rm -rf quote-calculator/

    Очистка данных (без удаления проекта)¶

    # Удалить базу данных
rm db/quotes.db

# Удалить legacy файлы
rm -rf estimate/*
rm -rf catalog/*
rm -rf backup/*

# Удалить логи
rm -rf logs/*

# Перезапустить сервер (создаст пустую БД)
npm start

    Типичные проблемы¶

    Порт уже используется¶

    Ошибка: 

    Error: listen EADDRINUSE: address already in use :::4000

    Решение: 

    # Найти процесс
lsof -i :4000

# Убить процесс
kill -9 <PID>

# Или использовать другой порт
PORT=3000 npm start

    Нет прав на запись¶

    Ошибка: 

    Error: EACCES: permission denied, open 'db/quotes.db'

    Решение: 

    # Дать права на запись
chmod -R 755 db/
chmod 644 db/quotes.db

# Или запустить с sudo (не рекомендуется)
sudo npm start

    Устаревшая версия Node.js¶

    Ошибка: 

    SyntaxError: Unexpected token '?'

    Решение: 

    # Обновить Node.js до версии 16+
# Для macOS (Homebrew)
brew upgrade node

# Для Windows
# Скачать установщик с nodejs.org

# Проверить версию
node --version

    npm install падает¶

    Ошибка: 

    npm ERR! code ECONNRESET

    Решение: 

    # Очистить cache
npm cache clean --force

# Попробовать снова
npm install

# Или использовать --legacy-peer-deps
npm install --legacy-peer-deps

    Логи и отладка¶

    Просмотр логов¶

    # Все логи
tail -f logs/combined.log

# Только ошибки
tail -f logs/error.log

# Последние 100 строк
tail -n 100 logs/combined.log

    Включение debug режима¶

    # В .env файле
LOG_LEVEL=debug

# Или через переменную окружения
LOG_LEVEL=debug npm start

    Проверка запросов¶

    # Мониторинг всех запросов
tail -f logs/combined.log | grep "HTTP"

# Мониторинг ошибок
tail -f logs/error.log
