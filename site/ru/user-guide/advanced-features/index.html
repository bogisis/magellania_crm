    Продвинутые функции¶

    Версия: 2.3.0 Дата обновления: 6 ноября 2025

    Введение¶

    Этот раздел описывает продвинутые возможности Quote Calculator для эффективной работы с большими объёмами данных и автоматизации рутинных задач.

    Содержание раздела¶

    1. Массовые операции¶

    Работа с множеством услуг одновременно: - Выбор нескольких услуг - Групповое изменение цен и наценок - Массовое удаление - Применение шаблонов

    Когда использовать: Повышение цен на сезон, изменение наценок по категориям, очистка старых данных

    Время чтения: 12 минут

    2. Шаблоны и пакеты¶

    Готовые наборы услуг для быстрого создания смет: - Создание шаблонов туров - Пакеты услуг - Переиспользование конфигураций - Библиотека стандартных туров

    Когда использовать: Типовые туры, повторяющиеся заказы, стандартизация предложений

    Время чтения: 10 минут

    3. Резервное копирование¶

    Защита данных и восстановление: - Автоматические backups - Ручное резервное копирование - Point-in-time recovery - Экспорт/импорт для миграции

    Когда использовать: Перед критическими изменениями, регулярные backups, миграция данных

    Время чтения: 15 минут

    Быстрые ссылки¶

    Частые задачи¶

    Задача Инструкция
    Повысить все цены на 10% Выбрать все (Ctrl+A) → Умножить цены × 1.10
    Изменить наценку для категории Фильтр по категории → Выбрать все → Установить наценку
    Создать backup перед изменениями Меню → Backup → Создать backup сейчас
    Восстановить из backup Меню → Backup → История → Восстановить
    Создать пакет услуг Выбрать услуги → Сохранить как пакет
    Применить шаблон тура Каталог → Пакеты → Выбрать → Добавить

    Горячие клавиши¶

    Действие Клавиша
    Выбрать все услуги Ctrl+A
    Снять выбор Escape
    Удалить выбранные Delete
    Создать backup Ctrl+Shift+B
    Применить изменения Ctrl+Enter

    Сценарии использования¶

    Сценарий 1: Сезонное повышение цен¶

    Задача: Повысить цены на все услуги на 15% к летнему сезону

    Шаги:
1. Создать backup текущих цен
   Меню → Backup → "Цены весна 2025"

2. Открыть каталог
   Ctrl+K

3. Выбрать все услуги
   Ctrl+A

4. Массовое изменение цен
   Кнопка "Цена" → "Умножить на 1.15"

5. Применить
   Ctrl+Enter

6. Проверить результат
   Сравнить несколько цен вручную

7. Сохранить
   Ctrl+S

    Время выполнения: 2-3 минуты Альтернатива: Ручное изменение каждой цены → 1-2 часа для 100 услуг

    Сценарий 2: Стандартизация наценок¶

    Задача: Установить единую наценку 12% для всех услуг категории "Размещение"

    Шаги:
1. Открыть каталог

2. Фильтр по категории
   Dropdown "Категория" → "accommodation"

3. Выбрать все отфильтрованные
   Ctrl+A (выбирает только видимые)

4. Установить наценку
   Кнопка "Наценка %" → Ввод: 12

5. Применить
   Все отели теперь с наценкой 12%

6. Сохранить

    Время выполнения: 1 минута Применимость: Любая категория, любая наценка

    Сценарий 3: Создание типового тура¶

    Задача: Создать шаблон "Бали 7 дней стандарт" для быстрого использования

    Шаги:
1. Открыть каталог "Бали"

2. Выбрать услуги для тура:
   ☑ Трансфер аэропорт-отель
   ☑ Трансфер отель-аэропорт
   ☑ Отель 4* (DBL, 6 ночей)
   ☑ Экскурсия обзорная
   ☑ Экскурсия храмы
   ☑ Гид (3 дня)

3. Сохранить как пакет
   Кнопка "Сохранить как пакет"
   Название: "Бали 7 дней стандарт"
   PAX по умолчанию: 10
   Наценка: 12%

4. Пакет создан

Использование:
  Новая смета → Каталог → Пакеты →
  "Бали 7 дней стандарт" → Добавить →
  Все услуги в смете с правильными параметрами

    Преимущества: - ✅ Создание сметы за 30 секунд вместо 10 минут - ✅ Гарантированная полнота (не забудешь услугу) - ✅ Консистентные наценки - ✅ Меньше ошибок

    Сценарий 4: Очистка старых данных¶

    Задача: Удалить неиспользуемые услуги из каталога

    Шаги:
1. Открыть каталог

2. Сортировка по использованию
   Dropdown "Сортировать" → "По популярности"

3. Фильтр
   Checkbox "Только с 0 использований"

4. Просмотр списка
   Проверить, действительно ли не нужны

5. Выбрать для удаления
   Checkbox у каждой ненужной

6. Удалить выбранные
   Кнопка "Удалить выбранные (N)"

7. Подтверждение
   Да → Backup создаётся автоматически

8. Готово
   Каталог очищен, размер уменьшился

    Рекомендуемая периодичность: Раз в квартал Безопасность: Automatic backup перед удалением

    Сценарий 5: Миграция на новый компьютер¶

    Задача: Перенести все данные на новую машину

    Подготовка (старый компьютер):
1. Полный backup
   Меню → Экспорт → Все данные
   Файл: full_backup_20250203.zip

2. Скопировать на USB или cloud
   Dropbox / Google Drive / USB флешка

Импорт (новый компьютер):
1. Установить Quote Calculator
   См. Installation Guide

2. Запустить приложение
   Проверить, что работает

3. Импорт backup
   Меню → Импорт → Полный импорт
   Выбрать: full_backup_20250203.zip

4. Подождать
   Импорт может занять 1-5 минут

5. Проверка
   - Открыть несколько смет
   - Проверить каталоги
   - Проверить расчёты

6. Готово
   Все данные на месте

    Что переносится: - ✅ Все сметы - ✅ Все каталоги - ✅ Все backups (опционально) - ✅ Настройки - ✅ История изменений

    Производительность¶

    Оптимизация для больших объёмов¶

    Рекомендуемые лимиты: 

    Оптимально:
  - До 100 услуг в массовой операции
  - До 1000 услуг в каталоге
  - До 500 смет в системе

Максимум (может тормозить):
  - 1000 услуг в операции
  - 5000 услуг в каталоге
  - 5000 смет в системе

    Если система тормозит¶

    Симптомы: - Долгая загрузка каталога (>5 секунд) - Задержка при массовых операциях - Медленное сохранение - UI фризы

    Решения:

    1. Разбить каталог 

      "Бали - Всё" (1500 услуг) →
"Бали - Трансферы" (150)
"Бали - Размещение" (800)
"Бали - Экскурсии" (300)
"Бали - Активности" (200)
"Бали - Прочее" (50)

    2. Архивировать старые сметы 

      Сметы старше 1 года →
Экспорт в ZIP →
Удалить из системы →
Хранить архив отдельно

    3. Оптимизировать базу данных 

      # SQLite vacuum
sqlite3 quotes.db "VACUUM;"

# Reindex
sqlite3 quotes.db "REINDEX;"

    4. Увеличить память (если возможно) 

      Chrome: chrome://settings → Производительность
Разрешить больше памяти

    Автоматизация¶

    Скрипты для повторяющихся задач¶

    Автоматическое повышение цен: 

    // price-update.js
const catalogs = await api.getCatalogs();
const multiplier = 1.10; // +10%

for (const catalog of catalogs) {
  const templates = catalog.data.templates;

  templates.forEach(t => {
    t.price = Math.round(t.price * multiplier * 100) / 100;
  });

  await api.updateCatalog(catalog.id, catalog);
  console.log(`✓ Updated ${catalog.name}`);
}

    Автоматический backup: 

    #!/bin/bash
# daily-backup.sh

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backups/daily"

# Backup через API
curl http://localhost:4000/api/export/all > "$BACKUP_DIR/backup_$DATE.json"

# Zip и загрузить в cloud
zip "$BACKUP_DIR/backup_$DATE.zip" "$BACKUP_DIR/backup_$DATE.json"
rclone copy "$BACKUP_DIR/backup_$DATE.zip" dropbox:QuoteCalc/Backups/

echo "✓ Backup completed: $DATE"

    Cron для ежедневного запуска: 

    # Добавить в crontab
crontab -e

# Backup каждый день в 3:00
0 3 * * * /path/to/daily-backup.sh

    API для продвинутых операций¶

    Массовое обновление через API¶

    Обновить все каталоги: 

    #!/bin/bash
# bulk-update-catalogs.sh

CATALOGS=$(curl -s http://localhost:4000/api/catalogs | jq -r '.catalogs[].id')

for id in $CATALOGS; do
  echo "Processing $id..."

  # Получить каталог
  catalog=$(curl -s "http://localhost:4000/api/catalogs/$id")

  # Обработать данные (например, через jq)
  updated=$(echo "$catalog" | jq '.data.templates |= map(.markup = 12)')

  # Обновить
  curl -X PUT "http://localhost:4000/api/catalogs/$id" \
    -H "Content-Type: application/json" \
    -d "$updated"

  echo "✓ Updated $id"
done

    Batch API (в разработке)¶

    Планируется в v2.4.0: 

    POST /api/batch
Content-Type: application/json

{
  "operations": [
    {
      "type": "update",
      "resource": "catalog",
      "id": "cat_001",
      "data": {...}
    },
    {
      "type": "delete",
      "resource": "estimate",
      "id": "est_123"
    }
  ]
}

Response:
{
  "completed": 2,
  "failed": 0,
  "results": [...]
}

    Безопасность продвинутых операций¶

    Перед массовыми изменениями¶

    Checklist: 

    □ Создан backup
□ Проверено на тестовых данных
□ Понятно, как откатить изменения
□ Есть время на проверку результата
□ Коллеги предупреждены (если работаете в команде)

    Автоматические защиты¶

    Система предотвращает: 

    // Удаление слишком большого количества
if (selectedCount > 100) {
    confirm(`Удалить ${selectedCount} услуг? Это много!`);
}

// Экстремальные изменения цен
if (multiplier > 2.0 || multiplier < 0.5) {
    confirm(`Изменение цен на ${(multiplier-1)*100}%? Проверьте!`);
}

// Удаление популярных услуг
if (service.usageCount > 50) {
    confirm(`"${service.name}" используется в ${service.usageCount} сметах. Удалить?`);
}

    Лучшие практики¶

    Работа с массовыми операциями¶

    ✅ Делайте: - Backup перед критическими изменениями - Тестируйте на маленькой выборке - Проверяйте результат перед сохранением - Документируйте сложные операции

    ❌ Не делайте: - Массовые операции без backup - Изменения без понимания последствий - Удаление без проверки использования - Игнорирование предупреждений системы

    Версионирование при больших изменениях¶

    Перед массовым обновлением:
1. Backup текущей версии
2. Создать новую версию каталога (v1.3.0)
3. Применить изменения к новой версии
4. Протестировать
5. Если OK → опубликовать
6. Если не OK → откатиться к v1.2.0
