    Управление каталогами¶

    Версия: 2.3.0

    Создание каталога¶

    Способ 1: С нуля¶

    1. Кнопка "Создать каталог"
2. Заполнить форму:
   Название: "Бали - Стандартные услуги"
   Регион: "Indonesia" (опционально)
   Описание: "Готовые услуги для туров на Бали"
3. Нажать "Создать"
4. Каталог создан (пустой)
5. Добавить услуги вручную

    Способ 2: Из импорта¶

    1. Кнопка "Импорт"
2. Выбрать JSON или CSV файл
3. Система валидирует
4. Каталог создаётся со всеми услугами
5. Готов к использованию

    Способ 3: Дублирование¶

    1. Открыть существующий каталог
2. Меню → Дублировать
3. Новое имя: "{original} - копия"
4. Можно изменить регион
5. Все услуги скопированы

    Редактирование метаданных¶

    Изменение названия¶

    1. Список каталогов
2. Клик на название
3. Инлайн редактирование
4. Ввод нового названия
5. Enter или клик вне поля
6. Автосохранение

    ⚠️ UNIQUE constraint: 

    Если (название + регион) уже существует:
  → Ошибка: "Каталог с таким названием уже есть"
  → Предложение: Изменить название или регион

    Изменение региона¶

    1. Клик на поле "Регион"
2. Dropdown со списком регионов:
   - Indonesia
   - Georgia
   - Russia
   - Thailand
   - USA
   - (Добавить свой)
3. Выбор региона
4. Автосохранение

    Добавление нового региона: 

    В dropdown → "Добавить регион"
Ввод названия: "Vietnam"
Регион добавлен в список
Доступен для других каталогов

    Изменение описания¶

    Клик на "Описание" → Textarea
Ввод текста (до 500 символов)
Ctrl+Enter для сохранения
Или клик вне поля

    Работа с услугами¶

    Добавление услуги¶

    Форма добавления: 

    Кнопка "+ Добавить услугу"

Обязательные поля:
  Название: "Трансфер аэропорт-отель"
  Цена: 25
  Количество: 1

Опциональные:
  Категория: transport
  Наценка %: 10
  Описание: "Седан, до 3 чел"
  Примечания: "Встреча с табличкой"

Кнопка "Добавить"

    Горячие клавиши: 

    Ctrl+Enter - Добавить и закрыть
Shift+Enter - Добавить и создать ещё
Escape - Отмена

    Редактирование услуги¶

    Инлайн редактирование: 

    Клик на любое поле услуги:
  - Название → Text input
  - Цена → Number input
  - Количество → Number input
  - Наценка → Number input
  - Категория → Dropdown

Enter - Сохранить
Escape - Отмена

    Форма редактирования: 

    Клик правой кнопкой → "Редактировать"
Или: Двойной клик на строку
Открывается полная форма
Все поля доступны
"Сохранить" или Ctrl+S

    Удаление услуги¶

    Способ 1: Наведи → Иконка 🗑️ → Клик
Способ 2: Выбрать → Delete key
Способ 3: Правый клик → Удалить

Подтверждение: "Удалить '{service}'?"
Да → Услуга удалена

    ⚠️ Удаление из каталога не удаляет из смет!

    Перемещение (сортировка)¶

    Drag & Drop:
  1. Захватить услугу (зажать ЛКМ)
  2. Перетащить вверх/вниз
  3. Отпустить
  4. Порядок сохранён

Кнопки [↑] [↓]:
  Клик → Услуга перемещается на 1 позицию

    Автосортировка: 

    Меню → Сортировка
  - По названию (A-Z)
  - По цене (возрастание)
  - По цене (убывание)
  - По категории
  - По использованию (популярные первыми)

    Массовые операции¶

    Выбор услуг¶

    Checkbox слева от услуги
Ctrl+Click - по одной
Shift+Click - диапазон
Ctrl+A - все услуги

    Индикатор выбора: 

    Панель вверху:
"Выбрано: 5 из 31 услуги"

    Массовое редактирование¶

    Изменить категорию: 

    1. Выбрать услуги
2. Кнопка "Категория" → Dropdown
3. Выбрать новую категорию
4. Применить ко всем выбранным

    Изменить наценку: 

    1. Выбрать услуги
2. Кнопка "Наценка %" → Input
3. Ввод значения: 12
4. Применить → Все выбранные теперь 12%

    Умножить цены: 

    Сценарий: Повышение цен на 10%

1. Выбрать все услуги (Ctrl+A)
2. Кнопка "Цена" → "Умножить на..."
3. Ввод: 1.10
4. Применить
5. Все цены увеличены на 10%

    Массовое удаление¶

    1. Выбрать услуги
2. Кнопка "Удалить выбранные (N)"
3. Подтверждение: "Удалить N услуг?"
4. Да → Все выбранные удалены

    Категории¶

    Стандартные категории¶

    ID Название Иконка
    transport Трансферы 🚗
    accommodation Размещение 🏨
    guide Гиды 👨‍🏫
    excursion Экскурсии 🏛️
    activity Активности 🎯
    food Питание 🍽️
    other Прочее 📋

    Создание категории¶

    Форма услуги → Категория → Dropdown
"+ Создать категорию"

Название: "Страхование"
ID: insurance
Иконка: 🛡️ (опционально)
Цвет: #3498db (опционально)

Создать → Категория доступна

    Управление категориями¶

    Меню → Настройки → Категории

Список всех категорий:
  - Переименовать
  - Изменить иконку
  - Изменить цвет
  - Удалить (если не используется)
  - Объединить с другой

    Удаление категории: 

    Если услуги используют эту категорию:
  → Нельзя удалить
  → Предложение: Сначала переназначить услуги

Если не используется:
  → Удаление разрешено

    Поиск и фильтрация¶

    Быстрый поиск¶

    Поле поиска в каталоге
Ввод: "трансфер"
Enter (или автопоиск после 300ms)

Результаты:
  - Трансфер аэропорт-отель
  - Трансфер отель-аэропорт
  - Трансфер между отелями

    Поиск по: - Названию услуги - Описанию - Примечаниям - Категории

    Регистронезависимый: 

    "ТРАНСФЕР" = "трансфер" = "Трансфер"

    Фильтры¶

    По категории: 

    Dropdown "Категория" → Выбор
Показываются только услуги этой категории
"Все категории" - сброс фильтра

    По цене: 

    Slider "Диапазон цен"
От: $0
До: $max (максимальная в каталоге)

Автообновление списка при изменении

    По популярности: 

    Checkbox "Только популярные"
Показываются услуги с > 10 использованиями

    Сортировка¶

    Dropdown "Сортировать по"
  - Названию (A-Z)
  - Названию (Z-A)
  - Цене (возрастание)
  - Цене (убывание)
  - Категории
  - Популярности
  - Дате добавления

    Версионирование¶

    Зачем нужно¶

    Проблема: 

    Изменили цены в каталоге
Старые сметы → Ссылаются на новые цены
Несоответствие в истории

    Решение: 

    Версионирование каталогов
Каждая смета → Snapshot услуг на момент создания
Изменения в каталоге не влияют на старые сметы

    Создание версии¶

    Автоматическое: 

    При критических изменениях:
  - Изменение большинства цен (>50%)
  - Удаление многих услуг (>10)
  - Изменение региона

Система предложит:
"Создать новую версию каталога?"
[Создать v1.3.0] [Продолжить v1.2.0] [Отмена]

    Ручное: 

    Меню → Версии → Создать новую
Номер: 1.3.0 (автоувеличение)
Описание: "Зимние цены 2025"
Создать

Старая версия (1.2.0) → Архив
Новая (1.3.0) → Активная

    Управление версиями¶

    Меню → Версии → История

┌─────────────────────────────────────┐
│ v1.3.0 (текущая)                    │
│ Зимние цены 2025                    │
│ 15 услуг изменено                   │
│ 3 февраля 2025                      │
├─────────────────────────────────────┤
│ v1.2.0                              │
│ Летние цены 2024                    │
│ 127 смет используют                 │
│ 1 июня 2024                         │
├─────────────────────────────────────┤
│ v1.1.0                              │
│ Расширение каталога                 │
│ 45 смет используют                  │
│ 15 января 2024                      │
└─────────────────────────────────────┘

    Действия: 

    Для старых версий:
  - Просмотр (read-only)
  - Экспорт
  - Сравнение с текущей
  - Восстановление (→ новая версия)
  - Удаление (если не используется в сметах)

    Шаблоны и пакеты¶

    Что такое пакет¶

    Пакет - готовый набор услуг для типового тура:

    {
  id: "pkg_bali_standard_7d",
  name: "Бали - Стандартный тур 7 дней",
  services: [
    "svc_transfer_airport_hotel",
    "svc_hotel_4star_dbl_6nights",
    "svc_tour_ubud",
    "svc_tour_temples",
    "svc_guide_3days"
  ],
  pax: 10,
  markup: 12
}

    Создание пакета¶

    1. Выбрать услуги в каталоге (checkbox)
2. Кнопка "Сохранить как пакет"
3. Название: "Стандартный тур 7 дней"
4. PAX по умолчанию: 10
5. Наценка по умолчанию: 12%
6. Сохранить

    Использование: 

    При создании сметы:
  Каталог → Вкладка "Пакеты"
  Выбрать пакет
  Все услуги добавляются в смету
  PAX и наценки применяются

    Управление пакетами¶

    Список пакетов в каталоге
Действия:
  - Редактировать состав
  - Изменить PAX/наценки
  - Переименовать
  - Дублировать
  - Удалить

    Резервные копии¶

    Автоматические backups¶

    Backup создаётся при:
  - Удалении услуги
  - Массовом изменении
  - Изменении версии
  - Импорте (перезаписи)

    Хранение: 

    Таблица catalog_backups
  - catalog_id
  - data (полный JSON)
  - timestamp
  - reason ("mass_delete", "import", etc.)

Автоудаление: Через 90 дней

    Восстановление¶

    Меню → Backups

Список:
  [3 фев 2025 15:30] Перед массовым удалением
  [1 фев 2025 10:15] Перед импортом
  [25 янв 2025 14:00] Версия 1.2.0

Действия:
  - Просмотр (сравнение с текущим)
  - Восстановить
  - Экспорт
  - Удалить backup

    Восстановление: 

    Выбрать backup → Кнопка "Восстановить"
Предупреждение: "Текущая версия будет заменена"
[Создать backup текущей] [Восстановить] [Отмена]

После подтверждения:
  1. Backup текущей версии создан
  2. Данные из backup восстановлены
  3. Уведомление: "Каталог восстановлен"

    Удаление каталога¶

    Процесс¶

    1. Список каталогов → Иконка 🗑️
2. Подтверждение: "Удалить каталог '{name}'?"
3. Проверка использования:

   Если используется в сметах:
     Предупреждение:
     "Каталог используется в 127 сметах.
      Удаление каталога НЕ удалит услуги из смет,
      но они больше не будут обновляться.
      Продолжить?"
     [Архивировать вместо удаления] [Удалить] [Отмена]

   Если не используется:
     [Удалить] [Отмена]

4. Финальное подтверждение:
   Ввод названия для подтверждения:
   "Введите '{name}' для подтверждения:"
   [________________]
   [Удалить]

5. Удаление:
   - Каталог удалён
   - Backup создан автоматически
   - Уведомление: "Каталог удалён"

    Что происходит¶

    Удаляется:
  ✅ Запись в таблице catalogs
  ✅ Все связи

НЕ удаляется:
  ❌ Услуги в сметах (они независимы)
  ❌ Backups (хранятся 90 дней)
  ❌ Статистика использования

    Восстановление удалённого¶

    В течение 90 дней:
  Меню → Backups → Удалённые каталоги
  Выбрать каталог
  Восстановить → Каталог создаётся заново
  Все услуги на месте

После 90 дней:
  Backup автоудалён
  Восстановление невозможно
  Нужен экспорт (если был сделан)

    Организация¶

    По географии + тип: 

    Бали - Стандартные услуги
Бали - Премиум услуги
Бали - Активный отдых

    По сезонам: 

    Сочи - Лето 2025
Сочи - Зима 2024-2025

    Обновление цен¶

    Рекомендуется:
  1. Создать новую версию
  2. Обновить цены
  3. Протестировать на новой смете
  4. Если OK → Опубликовать
  5. Старая версия → Архив

Не рекомендуется:
  ❌ Менять текущую версию без backup
  ❌ Менять цены во время активной работы
  ❌ Не информировать коллег об изменениях
