    Работа с каталогами¶

    Версия: 2.3.0 Дата обновления: 6 ноября 2025

    Введение¶

    Каталоги - это библиотеки готовых услуг и пакетов для быстрого создания смет. Система поддерживает множество каталогов для разных регионов и типов туров.

    Что такое каталог¶

    Определение¶

    Каталог - это именованная коллекция шаблонов услуг со следующими параметрами:

    {
  id: "cat_001",
  name: "Бали - Стандартные услуги",
  region: "Indonesia",
  version: "1.2.0",
  templates: [
    {
      id: "svc_001",
      name: "Трансфер аэропорт-отель",
      price: 25,
      quantity: 1,
      markup: 10,
      category: "transport"
    },
    // ... другие услуги
  ]
}

    Назначение¶

    • ✅ Скорость - не нужно вводить данные вручную
    • ✅ Консистентность - одинаковые названия и цены
    • ✅ Шаблоны - готовые пакеты для типовых туров
    • ✅ Мультирегиональность - отдельные каталоги по странам
    • ✅ Совместная работа - экспорт/импорт для обмена

    Работа с каталогами¶

    Открытие каталога (Ctrl+K)¶

    1. Нажать Ctrl+K (или кнопку "Каталог")
2. Выбрать каталог из списка
3. Браузер услуг открывается
4. Выбрать нужные услуги
5. Клик "Добавить в смету"

    Переключение между каталогами¶

    В окне каталога:
  Dropdown "Выбрать каталог" → Список всех каталогов
  Клик на название → Загрузка каталога
  Услуги обновляются автоматически

    Фильтрация и поиск¶

    Поиск по названию: 

    Поле "Поиск" → Ввод текста → Enter
Поиск по названию, описанию
Регистронезависимый

    Фильтр по категории: 

    Dropdown "Категория" → Выбор категории
Показываются только услуги выбранной категории
"Все категории" - сбросить фильтр

    Фильтр по цене: 

    Slider "Диапазон цен"
От $0 до максимальной в каталоге
Автообновление списка

    Добавление в смету¶

    Одиночное добавление: 

    1. Клик на услугу в списке
2. Кнопка "Добавить" (или Enter)
3. Услуга добавляется в текущую смету

    Множественное добавление: 

    1. Checkbox слева от услуги
2. Ctrl+Click для выбора нескольких
3. Кнопка "Добавить выбранные (N)"
4. Все выбранные добавляются

    Добавление с изменением: 

    1. Клик на услугу
2. Изменить количество/наценку
3. Кнопка "Добавить"
4. В смету добавляется с изменениями

    Мультирегиональность¶

    Зачем нужны регионы¶

    Проблема: - Услуга "Трансфер" в Бали стоит $25 - Та же услуга в Тбилиси стоит $15 - В Сочи - 1500₽

    Решение: 

    Каталоги с привязкой к региону:
  "Бали - Стандартные услуги" (region: "Indonesia")
  "Тбилиси - Корпоративные туры" (region: "Georgia")
  "Сочи - Летний отдых" (region: "Russia")

    UNIQUE constraint¶

    База данных: 

    CREATE TABLE catalogs (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    region TEXT,
    UNIQUE(name, region)  -- Один "Бали - Услуги" только для Indonesia
);

    Что это даёт: - ✅ Нельзя создать два каталога с одинаковым именем и регионом - ✅ Можно создать "Стандартные услуги" для Бали и для Тбилиси - ✅ Автоматическая защита от дублей

    Работа с регионами¶

    При создании каталога:
  Название: "Стандартные услуги"
  Регион: "Indonesia"

При импорте:
  Если (name + region) существует → CONFLICT
  Действие: Перезаписать? Переименовать? Отмена?

При экспорте:
  JSON содержит поле "region"
  При импорте на другой машине регион сохраняется

    Управление каталогами¶

    Создание нового¶

    1. Кнопка "Создать каталог"
2. Название: "Новый каталог"
3. Регион: (опционально)
4. Описание: (опционально)
5. "Создать"

    Начинается пустым, добавляйте услуги вручную или импортируйте.

    Редактирование¶

    Метаданные: 

    Клик на название каталога → Редактировать
Изменить: название, регион, описание
Сохранить

    Услуги: 

    Открыть каталог → Режим редактирования
Добавить услугу
Изменить существующую
Удалить ненужную
Сохранить изменения

    Удаление¶

    Список каталогов → Клик на иконку 🗑️
Подтверждение: "Удалить каталог '{name}'?"
Да → Каталог удалён (НЕ ВОССТАНОВИТЬ)

    ⚠️ Удаление каталога не удаляет услуги из смет!

    Дублирование¶

    Список каталогов → Меню → Дублировать
Новое имя: "{original} - копия"
Все услуги копируются
Можно изменить регион

    Использование: - Создать вариант каталога для другого региона - Экспериментировать с ценами - Сезонные версии (лето/зима)

    Шаблоны услуг¶

    Структура шаблона¶

    {
  id: "svc_001",                     // Уникальный ID
  name: "Трансфер аэропорт-отель",  // Название
  price: 25,                         // Цена за единицу ($)
  quantity: 1,                       // Количество по умолчанию
  markup: 10,                        // Наценка % (по умолчанию)
  category: "transport",             // Категория
  description: "Седан, до 3 чел",   // Описание (опц.)
  notes: "Включает встречу с табличкой" // Примечания (опц.)
}

    Обязательные поля¶

    • ✅ name - название услуги
    • ✅ price - цена (≥ 0)
    • ✅ quantity - количество (≥ 1)

    Опциональные поля¶

    • markup - наценка % (default: 0)
    • category - категория (default: "other")
    • description - краткое описание
    • notes - подробные примечания

    Валидация¶

    // Автоматическая проверка
if (price < 0) → Ошибка: "Цена не может быть отрицательной"
if (quantity < 1) → Ошибка: "Количество должно быть ≥ 1"
if (name.length > 100) → Ошибка: "Название слишком длинное"
if (markup > 100) → Предупреждение: "Подозрительно высокая наценка"

    Импорт и экспорт¶

    Форматы¶

    JSON (рекомендуется): 

    {
  "version": "1.2.0",
  "name": "Бали - Стандартные услуги",
  "region": "Indonesia",
  "templates": [
    {
      "id": "svc_001",
      "name": "Трансфер",
      "price": 25,
      "quantity": 1,
      "markup": 10,
      "category": "transport"
    }
  ]
}

    CSV (legacy): 

    name,price,quantity,markup,category
"Трансфер",25,1,10,"transport"
"Отель",80,6,10,"accommodation"

    Экспорт каталога¶

    1. Открыть каталог
2. Меню → Экспорт
3. Формат: JSON или CSV
4. Сохраняется файл: {название}_catalog.json
5. Можно отправить коллегам

    JSON экспорт содержит: - ✅ Все услуги - ✅ Метаданные (версия, регион) - ✅ Категории и описания

    CSV экспорт содержит: - ✅ Только основные поля (name, price, quantity, markup) - ❌ Без метаданных

    Импорт каталога¶

    1. Кнопка "Импорт"
2. Выбрать файл JSON или CSV
3. Система валидирует:
   - Формат файла
   - Версию
   - Структуру данных
   - UNIQUE(name, region) constraint
4. Если конфликт → Диалог:
   "Каталог '{name}' (region: '{region}') уже существует"
   [Перезаписать] [Переименовать] [Отмена]
5. Импорт завершён

    Валидация при импорте: 

    ✅ Проверка версии (поддерживается ли)
✅ Обязательные поля (name, price, quantity)
✅ Типы данных (price - число, quantity - целое)
✅ Диапазоны (price ≥ 0, quantity ≥ 1)
⚠️ Предупреждение если markup > 50%
❌ Ошибка если несовместимый формат

    API для каталогов¶

    Основные эндпоинты¶

    # Получить все каталоги
GET /api/catalogs

# Получить каталог по ID
GET /api/catalogs/:id

# Создать каталог
POST /api/catalogs
Content-Type: application/json
{
  "name": "Новый каталог",
  "region": "Indonesia",
  "templates": [...]
}

# Обновить каталог
PUT /api/catalogs/:id

# Удалить каталог
DELETE /api/catalogs/:id

    Подробнее: API Reference - Catalogs

    // Рекомендованные значения в шаблонах

category: "food"          → markup: 8%
category: "excursion"     → markup: 15%
category: "activity"      → markup: 18%

    Версионирование¶

    При изменении цен → Создать новую версию
Старые сметы → Ссылаются на старые цены
Новые сметы → Используют актуальные

Пример:
  "Бали - 2024" → архив
  "Бали - 2025" → актуальный

    1. Разбить на подкаталоги по категориям

2. Удалить неиспользуемые услуги
   Статистика → Сортировка по использованию → Удалить с 0 использований

3. Архивировать старые каталоги
   "Бали - 2023" → Экспорт → Удалить из системы
