    Начало работы¶

    Версия: 2.3.0 Дата обновления: 6 ноября 2025

    Добро пожаловать в Quote Calculator!¶

    Quote Calculator v2.3.0 - это профессиональная система для создания коммерческих предложений в туристическом бизнесе. Эта глава поможет вам быстро освоить основы работы с системой.

    Что вы узнаете¶

    В этом разделе вы научитесь:

    • ✅ Создавать сметы с нуля - за 5 минут
    • ✅ Работать с интерфейсом - все элементы управления
    • ✅ Использовать горячие клавиши - ускорить работу
    • ✅ Настраивать наценки и расчёты - правильная маржа
    • ✅ Сохранять и экспортировать данные - не потерять работу

    Быстрый старт¶

    5-минутное введение¶

    1. Запустите приложение 

      npm start
# или
docker-compose up -d

    2. Откройте в браузере 

      http://localhost:4000

    3. Создайте первую смету

    4. Заполните данные клиента
    5. Укажите количество человек (PAX)
    6. Добавьте услуги из каталога
    7. Настройте наценки
    8. Сохраните и распечатайте

    Готово! Вы создали свою первую смету.

    Основные концепции¶

    1. Смета (Estimate)¶

    Смета - это коммерческое предложение для клиента, содержащее: - Информацию о клиенте и группе - Список услуг с ценами - Расчёт итоговой стоимости - Наценки и маржу

    2. Каталог (Catalog)¶

    Каталог - это библиотека шаблонов услуг: - Готовые услуги с базовыми ценами - Категории (трансферы, отели, экскурсии) - Описания и примечания - Быстрое добавление в смету

    3. Наценки (Markup)¶

    Два типа наценок:

    Индивидуальная наценка (на каждую услугу): 

    Базовая цена: $100
Наценка 10%:  $10
Клиенту:      $110

    Скрытая наценка (общая на весь тур): 

    Базовая стоимость: $1000
Скрытая наценка 5%: $50
Итого прибыли:      $50 + индивидуальные

    ⚠️ Важно: Скрытая наценка НЕ показывается клиенту в печатной версии.

    4. PAX¶

    PAX (Passenger) - количество туристов в группе.

    Влияет на расчёт услуг: - Трансферы: цена × кол-во трансферов - Отели: цена × (PAX ÷ 2) для DBL номеров - Экскурсии: цена × группа (фиксированная) - Питание: цена × PAX × дни

    Типичные рабочие процессы¶

    Workflow 1: Стандартный тур-пакет¶

    1. Клиент запрашивает стандартный тур
2. Открываем приложение
3. Заполняем данные клиента (ФИО, контакты)
4. Указываем PAX и даты
5. Выбираем готовый пакет из каталога
6. Корректируем цены под сезон/спрос
7. Добавляем стандартную наценку 10-15%
8. Сохраняем и отправляем клиенту PDF

    Время: 3-5 минут

    Workflow 2: Индивидуальный тур¶

    1. Клиент описывает желания
2. Создаём новую смету
3. Заполняем данные клиента
4. Указываем PAX и примерные даты
5. Подбираем услуги вручную или из каталога
6. Добавляем custom услуги по запросу
7. Настраиваем индивидуальные наценки
8. Добавляем скрытую маржу 5-10%
9. Сохраняем как черновик
10. Согласовываем с клиентом
11. Корректируем и отправляем финальную версию

    Время: 10-15 минут

    Workflow 3: Корректировка существующей сметы¶

    1. Загружаем смету из списка (Ctrl+F)
2. Клиент просит изменения
3. Корректируем услуги/цены/даты
4. Пересчитываем автоматически
5. Сохраняем под новым именем или перезаписываем
6. Отправляем обновлённый PDF

    Время: 1-2 минуты

    Расчёты¶

    Как работают расчёты?¶

    1. Базовая стоимость
   = Сумма всех услуг (цена × количество)

2. Индивидуальные наценки
   = Для каждой услуги отдельно (цена × наценка%)

3. Скрытая наценка
   = % от БАЗОВОЙ стоимости (не показывается клиенту)

4. НДС
   = % от суммы БЕЗ скрытой наценки

5. Итого клиенту
   = Базовая + индивидуальные наценки + НДС

6. Ваша прибыль
   = Индивидуальные наценки + скрытая наценка

    Пример расчёта¶

    Услуги:
- Трансфер: $100 (наценка 10%)
- Отель:    $2,400 (наценка 10%)
- Экскурсии: $850 (наценка 15%)

Скрытая наценка: 5%
НДС: 0%

Расчёт:
1. Базовая стоимость = $100 + $2,400 + $850 = $3,350

2. С индивидуальными наценками:
   - Трансфер: $100 × 1.10 = $110
   - Отель: $2,400 × 1.10 = $2,640
   - Экскурсии: $850 × 1.15 = $977.50
   Итого: $3,727.50

3. Скрытая наценка (для внутренних расчётов):
   $3,350 × 0.05 = $167.50

4. Итого клиенту: $3,727.50
   Ваша прибыль: $377.50 + $167.50 = $545
   Маржа: 16.3%

    Сохранение¶

    Автосохранение¶

    Смета автоматически сохраняется в localStorage браузера каждые 30 секунд.

    ⚠️ Важно: Автосохранение локальное - данные не отправляются на сервер.

    Ручное сохранение¶

    На сервер (рекомендуется): 

    Нажмите "Сохранить" (Ctrl+S)
→ Смета сохраняется в базе данных
→ Создаётся автоматический backup

    В JSON файл: 

    Меню → Экспорт → JSON
→ Скачивается файл на компьютер

    Backup¶

    Система автоматически создает backup при каждом сохранении:

    Сохранение → estimate + backup

    Вы можете восстановить любую предыдущую версию: 

    Меню → Backups → Выбрать backup → Восстановить

    Печать¶

    Подготовка к печати¶

    1. Проверьте все данные
    2. Убедитесь, что скрытые поля не показываются
    3. Нажмите "Печать" (Ctrl+P)

    Что попадет в печать¶

    Показывается клиенту: - ✅ Информация о клиенте и группе - ✅ Список услуг с итоговыми ценами - ✅ Общая стоимость тура - ✅ Условия и примечания

    НЕ показывается клиенту: - ❌ Скрытая наценка - ❌ Внутренние расчёты прибыли - ❌ Базовые цены до наценок - ❌ Процент наценки

    Сохранение в PDF¶

    Печать → Сохранить как PDF → Отправить клиенту

    Частые вопросы¶

    Где хранятся мои данные?¶

    Три места: 1. localStorage - автосохранение в браузере (временно) 2. SQLite база данных - на сервере (постоянно) 3. Backups - автоматические резервные копии

    Потеряются ли данные при закрытии браузера?¶

    Нет, если вы сохранили на сервер (Ctrl+S). Автосохранение в localStorage может быть очищено браузером.

    Можно ли работать оффлайн?¶

    Частично. Frontend работает оффлайн, но сохранение на сервер требует интернета.

    Как отправить смету клиенту?¶

    Три способа: 1. Печать → Сохранить как PDF → Отправить email/мессенджер 2. Экспорт → JSON → Отправить файл 3. Печать → Распечатать на бумаге

    Можно ли работать с несколькими сметами одновременно?¶

    Нет, в текущей версии поддерживается работа только с одной сметой за раз. Но вы можете быстро переключаться между сметами через загрузку.
